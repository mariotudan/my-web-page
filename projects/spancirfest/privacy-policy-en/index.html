Privacy Policy

1. Introduction
This Privacy Policy governs the use of Špancirfest mobile application for devices with Android Lollipop or higher, which is created by Mario Tuđan.

This application provides information about Špancirfest. All information shown on a mobile application is also available on the Špancirfest website. Available information in the application is listed below:

- Program
- Interactive map with points of interest
- Image gallery from the official Špancirfest public album
- Quiz in cooperation with Tourist Board of Varaždin

2. Permissions while using the application
While using Špancirfest application, the application will need permissions for some feature. The user can choose to grant or deny permissions, but if permission is denied, some application functionalities may not be available. Permission needed for some application functionalities are:

- Permission to access the camera, to give a user a possibility to scan QR quiz codes
- Permission to write on external storage, to give user possibility to save images from the gallery and quiz results
- Permission to access user's location, to give user possibility to see user on map

3. Data collection
Špancirfest application can collect usage data while an application is in use and send crash reports, if the user agrees to enable those features. When an application is opened the first time, the user is asked if he wants to give permission to the application to collect usage data and send crash reports. The user can change that anytime in application settings.

3.1. Data usage collection
This application uses Firebase Analytics as a service for collecting usage data. The špancirfest application collects usage data and a consumption of content which is provided to the user, which also implies to get information about all content visited by the user inside the application.

At the first launch of an application, the user has a choice to give permission to the application to collect usage data. In case of giving the permission, the application will collect usage data, while if permission is not granted, data collection will be disabled. The user can change an option of data collection in application settings anytime.

Firebase Analytics automatically collects additional information while an application is in use, and those data are:

- Internet address
- Geolocation
- Device operating system
- Device model
- Number of users and sessions
- Session duration
- Unique device identifier
- First launches
- App opens
- App updates

Data is retained for 14 months, and after that is invalidated. If the user wants to delete its data earlier, this can be done by reset option in application settings.

3.2. Crash reporting
This application can send crash reports and system used for this functionality is Firebase Crashlytics. Automatic crash reporting gives developer detailed information about application issues and with that kind of reports, it's easier to detect and resolve common application issues which users are facing.

At the first launch of the application, the user has a choice to give permission to the application to send crash reports. In case of giving the permission, the application will send crash reports if one occurs, while if permission is not granted, crash reporting will be disabled. The user can change an option of crash reporting in application settings anytime.

Firebase Crashlytics collects data when sending crash reports, and those are:

- Installation UUID
- Crash traces

Crash traces and their associated identifiers are kept for 90 days.

4. Data usage
Špancirfest collects data of application usage to give a possibility to make the application better. Collected data gives insights into general usage od the application which leads to better usage understanding and to give users even better application experience.

Collected data will be used by Tourist Board of Varaždin for a better understanding of how the application is used in general and which content is most popular in general. Collected data will help Špancirfest in the future because it will give insights in content usage based on general users usage.

5. User choices and opt-out
At application launch, the user can choose to grant or deny to the application to collect usage data and to send crash reports. That choice is always available in application settings, and the user can change it anytime.

If the user wants to invalidate its collected data, in application settings is available reset option, which will invalidate all collected data from unique identifier used for data collection.

6. Agreement
By using Špancirfest mobile application, you agree to this Privacy Policy.
